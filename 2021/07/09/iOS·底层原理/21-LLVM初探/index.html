21-LLVM初探

一、前言

计算机是只能直接理解机器语言，而不能直接理解高级语言的，所以计算机要执行高级语言编写的程序，就必须要把高级语言翻译成机器语言。翻译有两种方式，一是编译，一是解释。所以不能被计算机直接识别的编程语言也可以分为解释型语言和编译型语言。

1.1 解释型语言

程序不需要提前编译，只需要在运行时使用专门的解释器对源程序逐行解释成特定平台的机器码并执行。一般用于脚本、辅助开发等。常见的解释型语言有Python、JavaScript、VBScript、Perl、Ruby、MATLAB等等。

特点：

  • 每次执行都需要将源代码逐行进行解释，效率较低

1.2 编译型语言

程序在执行之前需要一个专门的编译过程，使用专门的编译器，针对特定的平台，将源代码一次性的编译成可被该平台硬件执行的代码，并包装成该平台所能识别的可执行性程序的格式。在运行时不需要重新翻译，直接使用编译的结果即可。常见的编译型语言C、C++、Objective-C等等。

特点：

  • 在执行前有一个专门的编译过程，将编码编译为可执行文件
  • 运行时不需要编译，执行效率高

二、iOS指令架构

2.1 不同机型的指令集

真机的指令集：

  • iPhone5S及以上机型使用arm64指令集
  • 早期的机型使用的是armv7s、armv7

模拟器指令集：

  • 32位处理器是i386架构
  • 64位处理器是x86_64架构
  • M1 Chip 模拟器是arm64架构

2.2 Xcode中指令集相关

Build Setting 中：

  • Architectures：项目可支持哪些指令集类型。
  • Valid Architectures：限制可能被支持的指令集的范围。也就是Xcode编译出来的二进制包类型最终从这些类型产生，而编译出哪种指令集的包，则由Architectures与Valid Architectures的交集来确定。因此这个选项不能为空。
  • Build Active Architecture Only：指定是否只对当前连接设备所支持的指令集编译。当其值设置为YES，是为了debug的时候编译速度更快，它只编译当前设备的Architecture版本，而设置为NO时，会编译所有的版本。所以，一般debug的时候设置为YES，release的时候设置为NO。

三、一般编译器设计

简单的说，编译器就是将一种语言（通常为高级语言）翻译为另一种语言（通常为低级语言）的程序。

传统编译器设计：

3.1 编译器前端(Frontend)

解析源代码。它会进行：词法分析，语法分析，语义分析，检查源代码是否存在错误，然后构建抽象语法树(Abstract Syntax Tree，AST)，LLVM 的前端还会生成中间代码IR。

3.2 优化器(Optimizer)

优化器负责各种优化。改善代码的运行时间，例如消除冗余计算等。

3.3 后端(Backend)/代码生成器(CodeGenerator)

将代码映射到目标指令集，生成机器语言，并且进行机器相关的代码优化。

四、iOS 的编译器架构

Objective C/C/C++ 使用的编译器前端是 Clang，swift 是 Swift。后端都是 LLVM。LLVM 是编译器的框架系统，clang 只是编译器的前端部分。

五、LLVM概述

其他的编译器如 GCC，作为了一个整体的程序进行设计，所以不够灵活。
LLVM 最重要的设计就是，使用的通用的代码表示形式IR。所以 LLVM 可以灵活的增加前端和后端，只要增加的前端和后端使用同一的规则IR就可以。

六、Clang

Clang 属于 LLVM 架构中的前端，它的诞生是为了替代 GCC。

七、LLVM 整个编译流程

先说结论，整个编译流程包括：

0：输入原文件
1：预处理过程，包括宏的替换，头文件的导入等
2：编译阶段，词法分析、语法分析，最终生成IR代码
3：后端，代码优化处理，最终生成汇编
4：汇编生成目标文件
5：链接需要的动态库和静态库，生成可执行文件
6：通过不同的架构，生成对象的可执行文件

7.4 生成中间代码—IR

这是LLVM最独特的一点。语法分析之后，代码生成器就会根据语法树生成IR。

IR的语法：

  • @ 全局标识
  • % 局部标识
  • alloca 开辟空间
  • align 内存对齐
  • i32 4字节
  • store 写入内存
  • load 读取数据
  • call 调用数据
  • ret 返回
